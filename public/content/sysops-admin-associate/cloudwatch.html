By default, metrics are stored for 2 weeks. You can retrieve older metrics using the GetMetricStatistics API, or by using third party tools. You can retrieve data from any terminated EC2 or ELB instance for up to 2 weeks after its termination.

Default metric granularity is 5 minutes, detailed is 1 minute but costs more. For custom metrics, the minimum granularity that you can have is 1 minute.

Can set a CloudWatch alarm for any metric in account.

EC2 Status Checks

  • System Status - Checks physical machine hosting the instance
    • Loss of network connectivity
    • Loss of system power
    • Software issues on the physical host
    • Hardware issues on the physical host
    • Best way to resolve is to stop/start VM. This will start it on a different physical host.
  • Instance Status - Checks the virtual machine itself
    • Failed system status checks
    • Misconfigured networking or startup configuration
    • Exhausted memory
    • Corrupted file system
    • Incompatible kernel
    • Best way to troubleshoot is by rebooting or making modifications in OS
