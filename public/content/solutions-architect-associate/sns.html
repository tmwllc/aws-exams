SNS Tips

Delivers messages to clients using push, so clients do not need to poll for them. This is the opposite of SQS, which requires clients to poll for messages over and over. Can push notifications directly to mobile devices, but can also deliver them by SMS text message or email, to SQS queues or to any HTTP endpoint. Messages are stored redundantly across multiple availability zones. Deliveries message using JSON format.

Topics:

Allows you to group multiple recipients using topics. A topic is an access point for allowing recipients to dynamically subscripe for identical copies of the same notification. One topic can support deliveries to multiple endpoint types. For example, group together iOS, Android and SMS recipients. Publishing to a topic will deliver appropriately formatted copies to each subscriber.

  • HTTPS
  • Email

  • SQS
  • Application

Messages can be customized for each protocol

Subscriptions will not start sending messages until subscriptions are confirmed. When setting up a topic, if the subscription is not confirmed, it will expire in 3 days.

Messages will contain several key value pairs:

  • Type: Notification
  • MessagesId: guid
  • TopicArn
  • Subject
  • Message
  • Timestamp
  • SignatureVersion
  • Signature: encrypted value
  • SigningCertURL
  • UnsubscribeURL
  • MessageAttributes: [{}]
