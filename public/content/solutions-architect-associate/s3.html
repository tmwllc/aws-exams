S3 Tips

Security

  • By default all newly created buckets are private.
  • Access Control
    • Bucket policies - apply to entire bucket
    • Access Control Lists - can drill down to individual items within a bucket
  • Can be configured to create access logs which log all requests to the bucket. Can be logged to another bucket.

Encryption

  • In transit: SSL/TLS
  • At Rest:
    • Server Side Encryption
      • S3 Managed Keys - SSE-S3. Each object encrypted with unique key. Additionally, the key itself is encrypted with a amaster key, which is regularly reotated.
      • AWS Key Mangement Service, Managed Keys - SSE-KMS. Separate permissions for use of an envelope key. Envelope key is a key that protects the data's encryption key. Also provides audit trail of when keys were used and who used them. Can manage and encrypt keys yourself, or use default.
      • Server Side Encryption with Customer Provided Keys - SSE-C. Customer manages the encryption keys, Amazon manages the encryption/decryption.
    • Client Side Encryption
      • You encrypt data on client side and then upload it to S3.
