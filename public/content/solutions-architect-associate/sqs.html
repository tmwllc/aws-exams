SQS Exam Tips

  • Pull based, not pushed based
  • Messages are 256kb in size
  • Standard Queues and FIFO queues. FIFO queues are new and guarantee messages are processed in same order they are sent. Standard queues can come out of order.
  • Messages can be kept in queue from 1 minute to 14 days, default is 4 days
  • Visibility Time Out is the amount of time that the message is invisible in the SQS queue after a reader picks up that message
  • Provided the job is processed before the visibility timeout expires, the message will then be deleted from the queue
  • If the job is not processed within that time, the message will become visible again and another reader will process it. This could result in same message being delivered twice.
  • Because messages can be delivered twice, it's important to make processing of them idempotent.
  • Visibility Timeout maximum is 12 hours.
  • SQS Long Polling is a way to retrieve messages from SQS queues. While regular polling returns immediately, even if the message queue is empty. Long polling doesn't return a response until a message arrives in the queue, or the long poll times out.
  • Retention can be set to between 1 minute and 14 days
