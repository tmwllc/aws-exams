CloudWatch Tips

Can only see metrics for CPU, Disk, Network and Status (not Memory).

When creating an EC2 instance, it gives option for detailed CloudWatch monitoring. By default this is off, and regular monitoring will happen every 5 minutes. Detailed monitoring will happen every minute, but costs more.

Can setup alarm thresholds to trigger alerts when metrics exceed a defined level. Alerts can be sent via email.

Events help you respond to state changes in your AWS resources. For example, automatically invoke a Lambda to update DNS when an event notifies you that EC2 instances enters the running state.

Alarm Types

  • Alarm
  • Insufficient
  • OK

CloudWatch Logs can ingest, aggregate and monitor any text based common log data or JSON formatted logs.

To check free memory on a DB instance, look at Freeable Memory metric.

Retention Periods

  • 1 minute datapoints: 15 days
  • 5 minute datapoints: 63 days
  • 1 hour datapoints: 455 days
