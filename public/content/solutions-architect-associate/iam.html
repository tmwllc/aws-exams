Identity Access Management (IAM) Tips

New users are given a username/password as well as an access key id and secret access key. The username/password can only be used to login to the console, not for use with the CLI. The opposite is true of access key id and secret access key.

Federated users are users managed outside of AWS in your corporate directory.

Permission Types

  • Resource based
  • User based

It's always best to setup roles for permissions, even briefly needed ones. For example, a user needs to modify configuration of an ELB one time only. You should create an IAM role and attach a policy allowing modification access to the ELB.
